# 前端异常处理

# 错误类型

# Error

通用的错误构造函数,可以用于用户自定义错误类型的基类.除此之外,还有七种内置核心错误构造函数

  • EvalError
  • InternalError
  • RangeError
  • ReferenceError
  • SyntaxError
  • TypeError
  • UriError

# GlobalEventHandlers

描述了一系列web worker的事件接口 HTMLElement, Document, Window, 或 WorkerGlobalScope。这里面的每一个接口都可以添加更多的事件句柄。

# EventHandlers

这些事件定义在 GlobalEventHandlers 中, 并被混入和实现在HTMLElement, Document, Window上，任意html元素、document对象，window对象上均可以使用它提供的属性， 即时是全局的WorkerGlobalScope。

# onerror

混合事件 GlobalEventHandlers 的 onerror 属性是用于处理 error 的事件 Error事件的事件处理程序，在各种目标对象的不同类型错误被触发：

  • 运行时错误（包括：通过script引入的脚本中的语法错误），window会触发一个ErrorEvent接口的error时间，并执行window.onerror()
  • 当一项资源加载失败img/script，加载资源的元素会触发一个Event接口的error时间，并触发在该元素上onerror处理函数。这些error事件不会冒泡到window，但是可以被单一的window.addEventListener捕获

# tips

当引入的第三方跨域脚本中发生错误时，为了避免信息泄露，语法细节不会报告，取而代之的是简单的Script error.，通过script 的crossorigin属性并且服务器能下发适当的CORS HTTP Response Header ，这个行为就可以被覆盖。

# 几种异常捕捉方法及场景

# try...catch...

  • 用户捕捉运行时的异常
  • 无法捕捉语法错误，异步错误

# window.onerror

js错误时，window触发ErrorEvent接口的error事件，执行window.onerror来处理,return true后不往上抛异常

  • 同步运行时错误
  • 异步运行时错误
  • 可以捕捉语法错误（通过script引入的脚步中的语法异常）
  • 无法捕捉网络请求异常

# window.addEventListener('error',fn)

  • 可以捕获语法错误
  • 可以捕获远程资源加载异常(无法判断http请求的状态)

# Promise.catch

  • Promise中的异常

# window.addEventListener('unhandledrejection',fn)

  • Promise中的未被捕获的异常

# Vue中的异常捕获

  • Vue.config.errorHandler
